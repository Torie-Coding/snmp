# Development Guide

## Setup

```bash
# Clone the repository
git clone https://github.com/Torie-Coding/snmp.git
cd snmp

# Create a virtual environment (recommended)
python -m venv .venv
source .venv/bin/activate

# Install dev dependencies
pip install -r requirements-dev.txt
```

## Dependency Strategy

### Runtime

The only runtime Python dependency is `pysnmp>=6.2.0,<7.0.0`.

- Documented in `README.md` and module `DOCUMENTATION.requirements`.
- Enforced by a fail-fast `try/except ImportError` guard in the module.
- **Not** listed in `requirements-dev.txt` (which is dev-only), but is included there for convenience during development.
- There is no `requirements.txt` for runtime — runtime deps are installed by the user on the Ansible controller.

### Dev/Test

All development and testing dependencies are in `requirements-dev.txt`. This file is **not** shipped in the collection artifact (excluded via `build_ignore` in `galaxy.yml`).

## Running Tests

### Unit Tests

```bash
# Run unit tests with pytest
python -m pytest tests/unit/ -v
```

### Ansible Sanity Tests

```bash
# Requires the collection to be in the expected path
mkdir -p ansible_collections/torie_coding
ln -sf "$(pwd)" ansible_collections/torie_coding/snmp
cd ansible_collections/torie_coding/snmp
ansible-test sanity --docker -v
```

### Molecule Integration Tests

Molecule tests run the module against a real SNMP agent in Docker.

```bash
# Build and install the collection first
ansible-galaxy collection build --force
ansible-galaxy collection install torie_coding-snmp-*.tar.gz --force

# Run the default scenario
molecule test -s default
```

The Molecule default scenario:
1. Builds a Docker image with net-snmp (from `docker/Dockerfile.snmp-agent`)
2. Starts the SNMP agent container with v2c and v3 users
3. Runs `converge.yml` — exercises snmp_facts, snmp_get, and snmp_walk with v2c, v3/SHA, v3/SHA-256
4. Runs `verify.yml` — asserts all expected facts and results are returned
5. Tears down the container

### Integration Tests (ansible-test)

The `tests/integration/targets/snmp_facts/` target is provided for running with `ansible-test integration`. It requires a pre-existing SNMP agent. This is **not** run in CI (Molecule handles integration testing).

```bash
ansible-test integration snmp_facts -v
```

## Changelog

This collection uses [antsibull-changelog](https://github.com/ansible-community/antsibull-changelog) for changelog management.

### Committed Changelog Artifacts

| File | Purpose | When Created |
|---|---|---|
| `changelogs/config.yaml` | Configuration for antsibull-changelog | Once, at project init (committed) |
| `changelogs/fragments/*.yaml` | Individual change entries | Per PR/change (committed, consumed on release) |
| `changelogs/changelog.yaml` | Machine-readable changelog data | Generated by `antsibull-changelog release` (committed at release time) |
| `CHANGELOG.rst` | Human-readable changelog | Generated by `antsibull-changelog release` (committed at release time) |

### Adding Changelog Fragments

For each notable change, create a fragment file in `changelogs/fragments/`:

```yaml
# changelogs/fragments/add-snmp-facts-module.yaml
minor_changes:
  - snmp_facts - new module for gathering SNMP facts from network devices.
```

Fragment files are consumed (deleted) by `antsibull-changelog release` when the maintainer prepares a release. See [release.md](release.md) for the release workflow.

## Code Style

- Follow Ansible module development guidelines
- Use `module.fail_json()` for hard failures (auth, transport, timeout)
- Use `module.warn()` for non-fatal partial data issues
- No `host:port` parsing — `host` is always a bare hostname or IP
